Это пример постановки задачи  для бекендера и фронтендера на разработку страницы и ее функционала на основе прототипа, разработанного в текущем проекте (папке). dev tools не должны попадать в постановку задачи. путь к кликабельному прототипу берется из urk git hub, открывающего нужную страницу.


Саммари задачи



Реализовать в системе страницу «Перемещения» — журнал документов внутреннего перемещения материалов и реагентов между местами хранения. Каждый документ перемещения содержит шапку (откуда, куда, исполнитель, комментарий) и список позиций (номенклатура + лот + количество). Документ проходит жизненный цикл: черновик → проведён или отменён. Также требуется поддержка специального свойства места хранения/помещения «Списывать при перемещении».



Кликабельный прототип

https://maha5053.github.io/bt-prototype/sklad/peremeshcheniya



1. Изменения в справочниках «Места хранения», «Помещения»

Для справочников «Места хранения» и «Помещения» добавляется новое свойство «Списывать при перемещении» (булево, по умолчанию — не выбрано).

Если свойство включено, то при проведении документа перемещения в это место хранения/помещение перенесённый товар автоматически списывается: остаток на месте «Куда» не увеличивается, инициируется транзакция типа «Списание».



Важно: если свойство включено и у помещения, и у места хранения, создаётся ровно одна транзакция списания (не две).



2. Страница «Перемещения» — Журнал документов

Доступна для всех ролей.

2.1. Структура страницы

Страница состоит из:

заголовка «Перемещения»;

строки поиска;

кнопки «Добавить перемещение» (переход на форму создания);

кнопки «Фильтры» (модальное окно);

таблицы документов;

пагинации.

Каждая строка — один документ перемещения. Столбцы:

наименованию номенклатуры в позициях документа (частичное совпадение, по всем строкам).

Кнопка «Сбросить фильтры» — возвращает все фильтры в значение «Все» / пусто.

2.5. Сортировка

Таблица сортируется по номеру документа (по возрастанию / по убыванию). Направление переключается кликом по заголовку столбца.

Поведение:

При выборе места «Откуда» автоматически подгружаются доступные остатки (номенклатуры с положительным количеством).

При смене «Откуда» — все строки позиций сбрасываются.

Лот фильтруется по выбранной номенклатуре.

Количество блокируется, пока не выбран лот.

Кнопка «Добавить строку» — добавляет новую пустую строку. Неактивна, пока не выбрано «Откуда».

Кнопка удаления строки — удаляет строку. Неактивна, если строка последняя.

3.3. Валидация при создании

Если ни одна строка не заполнена (нет номенклатуры, лота, количества) — ошибка: «Добавьте хотя бы одну позицию с номенклатурой, лотом и количеством.»

Если в заполненной строке количество превышает доступный остаток — ошибка с указанием номенклатуры, лота и доступного количества.

Если место назначения настроено на автосписание — показывается диалог подтверждения (confirm):

«Место назначения «CNC» настроено на „Списывать при перемещении". После проведения документа перенесённый товар будет автоматически списан и не останется на остатках в этой зоне. Продолжить создание черновика?»

3.4. Результат создания

После успешного создания:

документ получает статус «черновик»;

номер — следующий по порядку;

фиксируется флаг "Списать при перемещении" (на основании свойства места «Куда»);

происходит переход на детальную страницу документа.

4. Детальная страница документа

Отображает полную информацию о документе перемещения.

4.1. Содержимое

Заголовок: номер документа.

Дата и время создания.

Статус (бейдж).

Бейдж «Автоматическое списание» — если место назначения со списанием.

Информационный блок — если автосписание включено: описание поведения.

Карточки: Откуда, Куда (с бейджем списания, если применимо), Исполнитель, Комментарий (если заполнен).

Таблица позиций: Номенклатура (ссылка на карточку), Лот, Количество.

4.2. Действия по документу

Доступны только для статуса «черновик»:

Кнопка

Действие

Провести

Статус → «проведено». Если автосписание — показывается alert с уведомлением.

Отменить

Статус → «отменено».

Для документов в статусе «проведено» или «отменено» кнопки действий не отображаются. Документ нельзя редактировать или удалять.



5. Логика проведения перемещения

При проведении документа перемещения:

Уменьшить остаток на месте хранения «Откуда» на количество из каждой позиции.

Увеличить остаток на месте хранения «Куда» на количество из каждой позиции.

5.1. Автоматическое списание

Если место «Куда» (или помещение, в котором оно находится) имеет свойство «Списывать при перемещении»:

шаг 2 (увеличение остатка на «Куда») не выполняется;

инициируется транзакция типа «Списание» на количество из каждой позиции;

в интерфейсе на детальной странице документа отображается информационный блок и бейдж «Автоматическое списание».

6. Хранение и нумерация

Документы перемещения хранятся в системе с уникальным id и сквозным порядковым number.

Нумерация — сквозная, не сбрасывается.

Каждый документ фиксирует createdAt — дату и время создания.

Исполнитель — пользователь, создавший документ (единственный ответственный).

7. Критерии приёмки

Пользователь может управлять свойством «Списывать при перемещении» в справочниках «Места хранения» и «Помещения».

Пользователь видит страницу «Перемещения» с журналом документов, поиском, фильтрацией (статус, откуда, куда) и пагинацией.

Пользователь может создать документ перемещения с выбором мест, позиций (номенклатура + лот + количество) и комментария.

При создании документа с местом назначения со свойством списания — пользователь получает предупреждение, флаг фиксируется в документе.

При проведении перемещения остатки уменьшаются на «Откуда» и увеличиваются на «Куда».

Если место «Куда» со свойством списания — остаток на «Куда» не увеличивается, инициируется транзакция «Списание».

Детальная страница документа отображает шапку, позиции, статус, бейдж автосписания и позволяет провести/отменить черновик.

Проведённые и отменённые документы нельзя редактировать или удалить.

Операции перемещения и списания отражаются в транзакциях и на детальной странице номенклатуры.
