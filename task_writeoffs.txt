Это задача для бекендера и фронтендера на реализацию страницы «Списания» и её функционала на основе прототипа, разработанного в текущем проекте (папке). dev tools не должны попадать в постановку задачи. Путь к кликабельному прототипу берется из URL GitHub, открывающего нужную страницу.

Саммари задачи

Реализовать в системе страницу «Списания» — журнал документов списания материалов и реагентов. Каждый документ списания содержит шапку (дата создания, создатель, статус, комиссия из 4 сотрудников, действие списания) и список позиций (номенклатура + лот + место хранения + количество + причина + комментарий). Документ проходит жизненный цикл: черновик → завершён. Завершение документа создаёт транзакцию списания, уменьшающую остатки на складе.

Кликабельный прототип

https://maha5053.github.io/bt-prototype/sklad/spisaniya

1. Модель данных «Документ списания»

Документ списания — это документ, содержащий шапку (дата создания, создатель, статус, комиссия, действие) и список списываемых позиций.

1.1. Структура документа списания

Поле | Тип | Описание
--- | --- | ---
id | string | Уникальный идентификатор документа
createdAt | datetime | Дата и время создания
createdBy | string | ФИО пользователя, создавшего документ
completedAt | datetime? | Дата и время завершения (null для черновика)
status | enum | "draft" / "completed"
commission | WriteOffCommission | Информация о комиссии (4 сотрудника)
action | enum | "utilized" (утилизировано) / "transferred" (передано)
lines | WriteOffLine[] | Список списываемых позиций

1.2. Структура комиссии (WriteOffCommission)

Поле | Тип | Описание
--- | --- | ---
headOfProduction | string | Руководитель производственной площадки (Зав. лабораторией)
headOfQuality | string | Руководитель ОКК
qualityEmployee | string | Сотрудник ОКК
authorizedPerson | string | Уполномоченное лицо

1.3. Структура позиции (WriteOffLine)

Поле | Тип | Описание
--- | --- | ---
nomenclatureId | string | Идентификатор номенклатуры
nomenclatureName | string | Наименование номенклатуры
group | string | Группа номенклатуры
manufacturer | string | Производитель
lot | string | Номер партии
place | string | Место хранения
quantity | number | Количество списываемого товара
reason | string | Причина списания (идентификатор из справочника причин)
comment | string | Комментарий к позиции

1.4. Справочник причин списания

ID | Метка
--- | ---
sterility | Нарушение условий стерильности
integrity | Нарушение целостности упаковки
expiry | Истечение срока годности
storage | Нарушение условий хранения
other | Иное

1.5. Справочник действий списания

ID | Метка
--- | ---
utilized | Утилизировано
transferred | Передано внутреннему поставщику

2. Страница «Списания» — Журнал документов

Доступна для всех ролей.

2.1. Структура страницы

Страница состоит из:

- заголовка «Списания»;
- строки поиска;
- кнопки «Создать списание» (создаёт новый документ и переходит на страницу детальной сессии);
- кнопки «Фильтры» (модальное окно);
- таблицы документов;
- пагинации.

2.2. Таблица документов

Каждая строка — один документ списания. Столбцы:

Столбец | Описание
--- | ---
Номер | Уникальный идентификатор документа (ссылка на детальную страницу)
Дата | Дата и время создания (формат ДД.ММ.ГГГГ ЧЧ:ММ)
Создал | ФИО пользователя, создавшего документ
Номенклатура | Превью первых 2 уникальных наименований из позиций документа. Если больше — показывается «+N». При наведении — тултип с полным списком
Лоты | Превью первых 2 уникальных номеров партий. Если больше — показывается «+N»
Общее кол-во | Сумма количеств всех позиций документа
Статус | Бейдж: «В процессе» (для черновика) / «Завершено» (для завершённого)

2.3. Поиск

Поле ввода. Ищет по:

- наименованию номенклатуры в позициях документа (частичное совпадение, по всем строкам);
- номеру партии (лоту);
- номеру документа (частичное совпадение);
- ФИО создателя (частичное совпадение).

2.4. Фильтрация (модальное окно)

Модальное окно с полями:

Фильтр | Тип | Значения
--- | --- | ---
Статус | Выпадающий список | Все / В процессе / Завершено

Кнопка «Сбросить фильтры» — возвращает все фильтры в значение «Все».
Кнопка «Закрыть» — закрывает модальное окно.
Кнопка «Фильтры» на странице подсвечивается (зелёная точка-индикатор), если хотя бы один фильтр активен (кроме статуса «Все»).

2.5. Пагинация

- Размер страницы: 15 строк.
- Навигация: «← Назад [номер страницы] Вперёд →».
- Отображение диапазона: «Показано X–Y из Z».
- Кнопка фильтров расположена в левой части футера пагинации.

2.6. Пустое состояние

Если документов нет (с учётом фильтров) — отображается сообщение: «Нет записей по фильтрам.» или «Нет документов списания.» (если фильтров нет).

3. Создание документа списания

При нажатии «Создать списание»:

1. Создаётся новый документ со статусом «draft».
2. Список позиций пуст.
3. Поле `action` устанавливается в `"utilized"` (по умолчанию).
4. Поле `commission` инициализируется пустыми значениями.
5. Происходит переход на детальную страницу документа.

4. Детальная страница документа списания

Отображает полную информацию о документе и позволяет добавлять/редактировать позиции, заполнять комиссию и выбирать действие.

4.1. Содержимое

- Заголовок: «Списание».
- Дата и время создания.
- Создатель (ФИО).
- Статус (бейдж: «В процессе» / «Завершено»).

4.2. Таблица позиций

Каждая строка содержит:

Столбец | Описание | Редактирование
--- | --- | ---
Номенклатура | Название + группа + производитель (3 строки) | Нет
Место хранения | Место хранения | Нет (указывается при добавлении)
Лот | Номер партии | Нет (указывается при добавлении)
Срок годности | Дата окончания срока годности | Нет (авто, из справочника номенклатуры)
Кол-во | Количество списываемого товара | Да (для черновика)
Причина | Причина списания | Да (для черновика, выпадающий список)
Комментарий | Текстовое поле | Да (для черновика)
Удалить | Кнопка удаления строки (×) | Да (для черновика)

4.3. Поведение ячеек «Кол-во»

- Отображается числовое значение.
- При клике — поле ввода (число, минимум 0).
- Сохранение по Enter или потере фокуса (blur).
- Отмена по Escape.
- Для завершённых документов ячейки нередактируемы — отображаются как plain text.

4.4. Поведение ячейки «Причина»

- Выпадающий список с 5 вариантами причин списания.
- Если причина не выбрана — поле отображается с оранжевой обводкой (визуальное предупреждение).
- Для завершённых документов — отображается метка причины (или «—», если не выбрана).

4.5. Поиск и фильтрация

На странице журнала (список документов) доступны поиск и фильтрация по статусу (см. раздел 2.3–2.4).
На детальной странице документа поиск и фильтрация позиций не предусмотрены.

4.6. Пагинация

- Размер страницы: 15 строк.
- Навигация: «← [номер страницы] →».
- Отображение диапазона: «X–Y из Z».

5. Добавление позиции в документ

Для документов в статусе «draft» доступна возможность добавления новых позиций.

5.1. Inline-форма добавления

При нажатии кнопки «Добавить позицию» под таблицей появляется строка с полями:

Поле | Тип | Обязательность | Комментарий
--- | --- | --- | ---
Номенклатура | Выпадающий список | да | Только номенклатуры с остатком > 0
Место хранения | Выпадающий список | да | Фильтруется по выбранной номенклатуре
Лот | Выпадающий список | да | Фильтруется по номенклатуре + месту хранения
Срок годности | Текст | нет | Автозаполнение из справочника (read-only)
Кол-во | Числовое поле | да | Минимум 1, максимум — доступный остаток
Причина | Выпадающий список | нет | 5 вариантов причин списания
Комментарий | Текстовое поле | нет | Произвольный текст

5.2. Каскадная логика выпадающих списков

1. Выбор номенклатуры → фильтруются места хранения, где есть остаток > 0 для данной номенклатуры.
2. Выбор места хранения → фильтруются лоты, доступные для данной номенклатуры в этом месте.
3. Выбор лота → автоматически заполняется срок годности из справочника.

5.3. Валидация количества

- Система проверяет доступный остаток по связке (номенклатура + лот + место хранения).
- Если введённое количество превышает доступный остаток — выводится ошибка: «Максимум: N».
- Под полем количества отображается подсказка: «Доступно: N».

5.4. Валидация при добавлении

- Обязательные поля: номенклатура, лот, количество.
- Если номенклатура или лот не выбраны — кнопка добавления (✓) неактивна.
- При успешном добавлении строка добавляется в таблицу, inline-форма закрывается.

5.5. Удаление позиции

- Каждая строка таблицы (кроме inline-формы) содержит кнопку удаления (×) — доступна только для черновиков.
- При нажатии строка удаляется без дополнительного подтверждения.

6. Комиссия

Блок «Комиссия» расположен под таблицей позиций.

6.1. Структура

Поле | Тип | Обязательность
--- | --- | ---
Руководитель производственной площадки (Зав. лабораторией) | Выпадающий список | нет
Руководитель ОКК | Выпадающий список | нет
Сотрудник ОКК | Выпадающий список | нет
Уполномоченное лицо | Выпадающий список | нет

6.2. Поведение

- Все 4 поля заполняются из справочника пользователей.
- Поля доступны для редактирования только в статусе «draft».
- Для завершённых документов — поля read-only.

7. Действие списания

Блок «Действие» расположен под блоком комиссии.

7.1. Структура

Поле | Тип | Значения
--- | --- | ---
Действие | Выпадающий список | «Утилизировано» / «Передано»

7.2. Поведение

- По умолчанию для новых черновиков: «Утилизировано».
- Доступно для изменения только в статусе «draft».
- Для завершённых документов — поле read-only.

8. Действия по документу

Доступны только для документов в статусе «draft».

8.1. Печать

- Кнопка «Печать» — открывает печатную форму акта списания в новом окне (popup).
- Печатная форма содержит:
  - Заголовок документа, дату, создателя.
  - Таблицу позиций (номенклатура, группа, производитель, место хранения, лот, срок годности, количество, причина, комментарий).
  - Блок комиссии (4 сотрудника).
  - Блок действия.
- Автоматический вызов `window.print()` при открытии.

8.2. Сохранить черновик

- Кнопка «Сохранить черновик» — явное сохранение текущего состояния.
- После нажатия отображается toast-уведомление (bottom-right):
  - Время сохранения (формат ЧЧ:ММ:СС).
  - Пример: `14:23:45`.
- Уведомление автоматически исчезает через 4 секунды.

8.3. Завершить документ

- Кнопка «Завершить» — переводит документ в статус «completed».
- При нажатии показывается диалог подтверждения:
  - «Завершить списание? Завершение создаст транзакцию списания, которая уменьшит остатки на складе. Действие нельзя отменить.»
- После подтверждения:
  - Статус → «completed».
  - Фиксируется `completedAt`.
  - Кнопки действий скрываются.
  - Все ячейки становятся нередактируемыми.
  - Отображается бейдж «Завершено».
  - Создаётся транзакция списания, уменьшающая остатки на складе.

8.4. Удалить черновик

- Кнопка «Удалить» (иконка корзины) — доступна только для черновиков.
- При нажатии показывается модальное окно подтверждения с красным предупреждением:
  - «Удалить списание? Черновик будет удалён безвозвратно.»
- После подтверждения документ удаляется, происходит переход на страницу журнала.
- Для завершённых документов кнопка удаления не отображается.

9. Хранение и нумерация

- Документы списания хранятся в системе с уникальным `id`.
- Нумерация документов — сквозная, порядковая.
- Каждая сессия фиксирует `createdAt` — дату и время создания.
- Создатель — пользователь, создавший документ.
- Данные документа сохраняются при каждом изменении (явное сохранение черновика или авто-сохранение при вводе).

10. Фронтенд — требования к интерфейсу

10.1. Список документов

- Таблица документов с колонками: номер, дата, создатель, номенклатура, лоты, общее кол-во, статус.
- Кнопка «Создать списание» — создаёт новый документ.
- Поиск по номенклатуре, лоту, номеру, создателю.
- Модальный фильтр по статусу с зелёной точкой-индикатором.
- Пагинация (15 строк на странице).
- Пустое состояние при отсутствии документов.
- Клик по строке — переход на детальную страницу.

10.2. Детальный документ

- Шапка с датой, создателем, статусом.
- Кнопка «Печать» — открывает popup с печатной формой.
- Таблица позиций с inline-редактированием количества и комментария.
- Выпадающий список причин списания для каждой позиции.
- Кнопка «Добавить позицию» — добавляет inline-строку с каскадными выпадающими списками.
- Кнопка удаления строки (×) для каждой позиции.
- Пагинация (15 строк на странице).
- Блок комиссии (4 выпадающих списка).
- Блок действия (выпадающий список: утилизировано/передано).
- Кнопки действий: «Печать», «Сохранить черновик», «Завершить», «Удалить» (только для draft).
- Toast-уведомление при сохранении черновика.
- Модальные окна подтверждения для завершения и удаления.

10.3. Стили и UX

- Редактируемые ячейки при клике превращаются в input с autofocus.
- Сохранение по Enter или blur, отмена по Escape.
- Незаполненная причина списания — оранжевая обводка выпадающего списка.
- Завершённые документы полностью read-only — без кнопок и редактирования.
- Inline-форма добавления позиции — каскадные выпадающие списки (номенклатура → место → лот).
- Под полем количества в inline-форме — подсказка «Доступно: N» и ошибка при превышении.
- Печатная форма — отдельное окно (popup), автовызов print.

11. Бекенд — требования к API

11.1. Эндпоинты

Метод | Путь | Описание
--- | --- | ---
GET | `/api/write-offs` | Получить список всех документов списания (краткая информация)
POST | `/api/write-offs` | Создать новый документ списания
GET | `/api/write-offs/{id}` | Получить детальную информацию о документе
POST | `/api/write-offs/{id}/lines` | Добавить позицию в документ
DELETE | `/api/write-offs/{id}/lines/{lineIndex}` | Удалить позицию из документа
PUT | `/api/write-offs/{id}/lines/{lineIndex}/quantity` | Обновить количество позиции
PUT | `/api/write-offs/{id}/lines/{lineIndex}/reason` | Обновить причину списания
PUT | `/api/write-offs/{id}/lines/{lineIndex}/comment` | Обновить комментарий к позиции
PUT | `/api/write-offs/{id}/commission` | Обновить информацию о комиссии
PUT | `/api/write-offs/{id}/action` | Обновить действие списания
POST | `/api/write-offs/{id}/complete` | Завершить документ списания
POST | `/api/write-offs/{id}/save-draft` | Явно сохранить черновик
DELETE | `/api/write-offs/{id}` | Удалить черновик (только статус draft)
GET | `/api/write-offs/{id}/print` | Получить данные для печатной формы

11.2. Бизнес-логика

- При создании документа список позиций пуст, `action` = `"utilized"`.
- Расчёт доступного остатка производится на сервере при добавлении позиции (номенклатура + лот + место хранения).
- При завершении документа создаётся транзакция списания, уменьшающая остатки на складе.
- Завершённый документ нельзя изменить или удалить.
- Удаление возможно только для документов в статусе «draft».

11.3. Валидация

- `quantity` ≥ 0.
- `quantity` не может превышать доступный остаток по связке (номенклатура + лот + место хранения).
- `comment` — макс. 500 символов.
- Завершение документа: статус должен быть «draft».
- Удаление: статус должен быть «draft».
- Причина списания — допустимый идентификатор из справочника.

12. Критерии приёмки

- Пользователь может создать новый документ списания с пустым списком позиций.
- Пользователь может добавлять позиции через inline-форму с каскадными выпадающими списками (номенклатура → место → лот).
- Срок годности автозаполняется из справочника при выборе лота.
- Количество валидируется по доступному остатку (номенклатура + лот + место хранения).
- Пользователь может редактировать количество и комментарий inline (Enter/blur для сохранения, Escape для отмены).
- Пользователь может выбрать причину списания из 5 вариантов.
- Пользователь может удалить позицию из документа (черновик).
- Пользователь может заполнить комиссию (4 сотрудника).
- Пользователь может выбрать действие списания (утилизировано/передано).
- Пользователь может сохранить черновик с toast-уведомлением (время).
- Пользователь может завершить документ с подтверждением — создаётся транзакция списания, уменьшающая остатки.
- Пользователь может удалить черновик с подтверждением — завершённые документы удалить нельзя.
- Пользователь может открыть печатную форму акта списания в popup с автовызовом print.
- Журнал документов отображает все документы со статусами, датами, превью номенклатуры и лотов.
- Завершённые документы полностью read-only.
- Пользователь может фильтровать документы по статусу и искать по номенклатуре, лоту, номеру, создателю.
- Пагинация работает (15 строк на странице).
