# Structure

**Analysis Date:** 2026-05-15

## Repository Root

- `package.json` - npm scripts and dependencies.
- `vite.config.ts` - Vite config, GitHub Pages base path, dev server/HMR/watch settings.
- `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json` - TypeScript project configuration.
- `eslint.config.js` - Flat ESLint config.
- `index.html` - SPA HTML shell.
- `public/` - Static assets copied into the Vite build.
- `src/` - Application source.
- `AGENTS.md` - Current agent operating instructions and session memory.
- `agents-done.md` - Longer feature/history memory.
- `.agents/skills/end-session/` - Local repo skill for session finalization.
- `.codex/` - Local GSD installation, ignored by git.
- `.planning/codebase/` - GSD codebase map generated by this workflow.

## Generated or Suspicious Root Artifacts

- `assets/index-D3cUk7Lf.js`, `assets/index--O1S8XbE.css`, and PDF copies under `assets/` look like built or copied output committed outside `dist`.
- `tsconfig.tsbuildinfo` is a generated TypeScript build cache.
- `task_example.txt`, `task_inventory.txt`, `task_writeoffs.txt` look like task notes or fixtures.

## Naming Conventions

- Page files usually use PascalCase and `Page`/`Pages` suffixes.
- Context files use `*Context.tsx`.
- Mock files use domain names plus `Data` or `Mock`.
- Russian transliteration appears in URL paths: `nomenklatura`, `postupleniya`, `inventarizatsiya`, `spisaniya`, `peremeshcheniya`, `proizvodstvo`, `polzovateli`.
